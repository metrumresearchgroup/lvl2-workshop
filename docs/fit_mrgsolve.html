Like in the warmup vignette, we create a function that takes care of getting the prediction and calculating the objective function (ols, here).

  • p parameters
  • .mod model object
  • .data the estimation data set
  • .yobs vector of observed data
  • pred if TRUE, just return a prediction
ols <- function(p, .mod, .data, .yobs, pred=FALSE) {
  
  p <- lapply(p, exp)
  
  .mod <- update(.mod, param=p)
  
  if(pred) {
    .data <- filter(.data, evid !=0)
    return(mrgsim(.mod,.data))
  }
  
  out <- mrgsim(.mod, data=.data, obsonly=TRUE)
  
  y.hat <- out$CP
  
  sum(c(y.hat-.yobs)^2)
}

The workflow is similar to the warmup vignette, but some important differences

  1. We have the parameters on log scale here, so we need to untransform
  2. We cannot use the parameters directly to get the prediction; we need to perform an update step to bring the parameter set being proposed by the optimizer into the model object
  3. We use mrgsim to actually generate the prediction
    • What would happen if we simulated with obsonly = FALSE?
  4. When we call ols with pred = TRUE, we can get the prediction

3 Speed up the simulation with qsim

ols.q <- function(p,.mod,.e,.data,.yobs, pred=FALSE) {
  
  p <- lapply(p, exp)
  
  .mod <- update(.mod,param=p)
  
  if(pred) {
    .data <- filter(.data, evid!=0)
    return(mrgsim(.mod,.data))
  }
  
  out <- qsim(.mod, .e)
  
  y.hat <- out[,"CP"]
  
  sum(c(y.hat-.yobs)^2)
}
datam <- recmatrix(as.ev(dose),times=obs$time)
yhat <- qsim(mod,datam)

4 The optimhelp package

This is in progress, but I almost always use it when working problems of any size.

We can make a set of parameters like this

library(optimhelp)


cl <- log_par("CL", 1)
vc <- log_par("VC", 10)

p <- parset(cl,vc)
p
.  name value transf tr fx
.    CL     1    log  u   
.    VC    10    log  u

So we have two parameters (CL and VC) with parameters (1 and 10) and they are both on log scale.
